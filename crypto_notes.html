密碼學口試重點筆記

一、對稱式加密（Symmetric Encryption）

定義：加密與解密使用同一把金鑰。

代表演算法：AES

優點：

  • 速度快
  • 適合大量資料加密
  • 計算成本低

缺點：

  • 金鑰分配困難
  • 不適合多方通訊

實務應用：通常搭配公鑰加密進行 Hybrid Encryption。

二、非對稱式加密（Public Key Encryption）

定義：使用公鑰加密、私鑰解密。

代表演算法：RSA

特性：

  • 解決金鑰分配問題
  • 計算較慢

實務應用：通常用來交換對稱式加密的 Session Key。

三、雜湊函數（Hash Function）

定義：將任意長度輸入轉為固定長度輸出，不可逆。

代表演算法：SHA-256

特性：

  • 不可逆
  • 固定輸出長度
  • Avalanche Effect

用途：

  • 密碼儲存（需加 salt）
  • 資料完整性驗證

四、數位簽章（Digital Signature）

目的：驗證身份 + 保證資料未被竄改。

流程：

  1. 對訊息做 Hash
  2. 用私鑰加密該 Hash
  3. 接收方用公鑰驗證

重點：簽章 ≠ 加密。簽章強調「認證與完整性」。

五、SSL / TLS 在做什麼？

功能：保護 Client 與 Server 之間的通訊安全。

提供三大安全性：

  • 機密性（加密傳輸）
  • 完整性（防止竄改）
  • 身份驗證（憑證機制）

基本流程：

  1. 伺服器提供數位憑證
  2. 使用公鑰交換 Session Key
  3. 後續通訊使用對稱式加密

注意：HTTPS ≠ 絕對安全，只保障傳輸安全。

六、加密 vs Hash 比較

項目 加密 Hash
可逆性 可逆 不可逆
是否需要金鑰 需要 不需要
主要用途 機密性 完整性

七、PostgreSQL 在全端會員系統中的加密機制

1️⃣ 傳輸加密（Client ↔ Server）

  • 使用 TLS（SSL）保護資料庫連線
  • 防止中間人攻擊（MITM）
  • 確保傳輸過程中的機密性與完整性

2️⃣ 會員密碼儲存方式

  • 不儲存明文密碼
  • 使用 SCRAM-SHA-256 機制
  • 包含 salt + challenge-response 驗證
  • 避免 rainbow table 攻擊

3️⃣ 應用層密碼保護（實務建議）

  • 後端通常使用 bcrypt 或 Argon2 進行密碼雜湊
  • 即使資料庫被入侵，攻擊者也難以還原密碼
  • 不可使用單純 SHA-256 直接儲存密碼

4️⃣ 資料本身是否加密？

  • 預設資料表內容不自動加密
  • 可使用 pgcrypto extension 進行欄位加密
  • 或使用磁碟層加密（如 LUKS）

口試重點：

PostgreSQL 提供傳輸加密與安全驗證機制， 但會員密碼保護主要應由應用層負責， 採用強雜湊演算法與適當的存取控制策略。

八、AES 與 RSA 簡明筆記

1️⃣ AES (Advanced Encryption Standard)

  • 類型：對稱式區塊加密
  • 加密/解密金鑰相同
  • 區塊大小：128 bits，金鑰長度：128/192/256 bits
  • 速度快，適合大量資料加密
  • 用途：HTTPS、磁碟加密、VPN、檔案加密
  • 核心概念：Substitution + Permutation + 混淆與擴散

2️⃣ RSA (Rivest-Shamir-Adleman)

  • 類型：非對稱式加密（公鑰/私鑰）
  • 加密用公鑰，解密用私鑰
  • 安全基礎：大整數質因數分解困難
  • 速度較慢，適合金鑰交換與數位簽章
  • 不適合加密大量資料

3️⃣ AES vs RSA 比較

項目 AES RSA
類型 對稱式 非對稱式
金鑰 同一把 公鑰 / 私鑰
速度 快 慢
用途 資料加密 金鑰交換、簽章
安全基礎 結構設計與輪函數 質因數分解困難

4️⃣ 實務應用

在 HTTPS 或混合式加密 (Hybrid Encryption) 中： RSA 用於交換 AES session key， AES 用於加密後續大量資料。

5️⃣ 口試簡答模板

AES 是對稱式區塊加密，用同一把金鑰加密/解密，速度快，適合大量資料； RSA 是非對稱式加密，使用公私鑰，安全性建立在質因數分解困難，通常用於金鑰交換與數位簽章；兩者實務上常結合使用。
